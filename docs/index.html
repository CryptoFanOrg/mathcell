MathCell
a simple straightforward way to include interactive mathematics in a web page

  • Why would I want to use MathCell?

    The ubiquity of web browsers provides an incredible opportunity to move beyond the static world of paper for the presentation of mathematics. The default language of the browser is JavaScript and modern presentations use that language. MathCell provides a quick way to set up interactive elements for input into your code, as well as outputs for visualizing the results of calculations.

  • How do I get MathCell?

    MathCell is free and open source, and the code is available on GitHub. You can download the complete library to your web server and include it in the body of a page with a script tag: 

    <script src="mathcell.js"></script>
  • How do I start a MathCell?

    First copy this template into the body of the page:

    <div class="mathcell" style="height: 4in">
<script>

var parent = document.scripts[ document.scripts.length - 1 ].parentNode;

var id = 'id' + document.scripts.length;
parent.id = id;

MathCell( id, [ { type: '' } ] );

parent.update = function( id ) {

  var name = +document.getElementById( id + 'name' ).value;

  ( additional JavaScript here )

  var data = [];

  var config = { type: '' };

  evaluate( id, data, config );

}

parent.update( id );

</script>
</div>

    The template automatically assigns an id to each cell, which is then available to the interactive input elements that are added to the cell. This relieves the end user of the need to manually add tags to cells.

    Input elements are specified in the dictionary inside the MathCell() command. Code for processing the input is added to the body of the update function. The output element is specified in the config dictionary inside the update function.

  • What interactive inputs are available?

    There are currently three kinds of inputs: slider, buttons and checkbox. The array of inputs in the MathCell() command should contain a dictionary for each input desired, with type set equal to one of these values followed by a comma-separated list of optional attributes.

    All inputs have two text options: a name that is used to identify the input in subsequent code, and a label that appears next to the input. The label can be any valid HTML. Single letter labels are italicized by default.

    A slider input has four numeric options with default values

    min0
    max1
    step.01
    defaultmin

    and the variable is assigned in code using

    var name = +document.getElementById( id + 'name' ).value;

    A buttons input has two additional options, one an array and the other numeric, with default values

    list[1,2,3]
    defaultlist[0]

    and the variable is assigned in code using

    var name = +document.querySelector( 'input[name=' + id + 'name]:checked' ).value;

    A checkbox input has a single additional option default, which can be set to either 'checked' or true to have the box initially checked. The variable is assigned in code using

    var name = document.getElementById( id + 'name' ).checked;

    IMPORTANT: for inputs that are intended to provide numeric values, be sure to include the plus signs appearing here. This unary operator invokes the Number object and ensures that the value will truly be numeric.

  • What visualization outputs are available?

    There are currently four kinds of inputs: svgplot, threejs, matrix and text. The config dictionary in the update function should have its type set equal to one of these values.

    An svgplot output displays a two-dimensional plot of the data. The input data should be an array of arrays of data points, i.e. a three-dimensional array. An svgplot has options and default values

    xMincalculated
    xMaxcalculated
    yMincalculated
    yMaxcalculated
    ticksauto
    axesLabelsnone
    colorsblack
    includeOriginfalse
    includeVerticalsfalse

    ticks can be set to an array with two numbers determining distances between ticks. colors is an array specifying the color of each array in the input data, with each given by standard HTML color codes.

    A threejs output displays a three-dimensional plot of the input data with the options and default values

    xMincalculated
    xMaxcalculated
    yMincalculated
    yMaxcalculated
    zMincalculated
    zMaxcalculated
    aspectRatio[1,1,1]
    axesfalse
    axesLabels['x','y','z']
    decimals2
    frametrue

    A matrix output displays the contents of a two-dimensional array in matrix form. You may need to adjust the width and height of the MathCell to show the entire array properly.

    A text output displays the result of a calculation in raw form. The data for this output can be a number, a string or an array as desired.

  • Why does my MathCell look strange?

    MathCell by default includes CSS styling to standardize the appearance of interactive elements across browsers. The one piece of this that is not set automatically is the height of the cell, since the number of inputs and the type of output is not known in advance. While this may be automated to some extent in the future, it is simpler to have the end user manually set the height in the style attribute on the first line of the cell as needed.

  • What if I want higher-level mathematics as well?

    A JavaScript library of special functions and common mathematical operations has been initiated, entitled simply Math. It provides higher-level mathematics not available by default in JavaScript.
